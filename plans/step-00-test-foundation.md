# Step 0 — Test & tooling foundation

Подготовить инфраструктуру для последующих шагов так, чтобы все дальнейшие шаги могли добавлять тесты без отдельной настройки.

## Работа

- Добавить тест-раннер (Vitest) в backend и в корневой workspace, npm-скрипт `pnpm test`.
- Настроить Fastify integration-тест helper: фабрика приложения без `listen()`, временная SQLite-база на файловой системе / `:memory:`, прогон Drizzle миграций перед каждым сьютом.
- Добавить минимальный smoke-тест «GET /health → 200» и соответствующий роут в backend, чтобы тесты проверяли живой конвейер.
- Обновить `AGENTS.md` командой `pnpm test` (если документация команд живёт там).

## TypeSpec / БД

В этом шаге ещё нет доменных сущностей — только `/health`, поэтому без БД.

## Состояние после шага

- `pnpm install`, `pnpm build`, `pnpm test` зелёные.
- Есть роут `/health`.
- Тестовый конвейер готов к использованию в следующих шагах.
