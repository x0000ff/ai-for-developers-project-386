# Step 2 — AdminPage: управление типами встреч

Владелец управляет типами встреч через UI.

## Работа

- Сгенерированный `openapi-typescript` клиент для `/event-types`.
- `AdminPage`:
  - Список типов (Mantine Table/Card).
  - Форма создания.
  - Редактирование in-place или в модалке.
  - Удаление с подтверждением.
- Обработка ошибок API, индикаторы загрузки.
- Мелкие компонентные тесты формы и вызовов клиента (моки fetch) — опционально, но желательно.

## TypeSpec / БД

Без изменений. Используем контракт из Step 1.

## Состояние после шага

`/admin` полностью управляет типами встреч, данные сохраняются и переживают перезапуск.
