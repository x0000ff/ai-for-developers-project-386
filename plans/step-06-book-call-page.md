# Step 6 — BookCallPage: выбор типа, календарь, слоты, форма

Гость проходит весь сценарий бронирования в UI.

## Работа

`BookCallPage` переписывается:

- Селектор типа встречи (предзаполнен из `?eventTypeId=`, если есть).
- Mantine `DatePicker` для выбора даты (future only).
- Загрузка слотов из `/event-types/{id}/slots` при смене типа/даты:
  - Индикатор загрузки.
  - Пустое состояние.
- Выбор слота, форма `guestName` + `guestEmail`, валидация.
- Сабмит `POST /bookings`:
  - 201 → экран подтверждения.
  - 409 → сообщение «слот только что заняли» и перезапрос слотов.
  - 400 / 404 → понятные ошибки.
- Переключение типа пересчитывает сетку (новая длительность).

## Тесты

Компонентные тесты ключевых состояний (список слотов, сабмит, обработка 409) с моками API-клиента.
